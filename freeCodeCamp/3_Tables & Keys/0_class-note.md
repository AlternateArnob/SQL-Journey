Primary key = a unique ID for each record(row) in a table.

- - - - - 

There are 2 kinds of primary keys 
1. surrogate key
2. natural key

- - - - - 

Surrogate key = a key that has no mapping to anything in the real world. 
(it is a type of primary key)

- - - - - 

Natural key = a key that has a mapping or has a purpose in the real world. 
(SSN = social security number)

- - - - - 

Foreign key = an attribute that will link us to another database table. 
- A foreign key stores the primary key of a row in another table in the same database. 
- A foreign key is just a way to define relationships between tables. 
Foreign key = A foreign key stores the value of the primary key of another table. 

- A table can have more than 1 foreign key on it. 
- A foreign key can reference the same table — this is called a (self-referencing)foreign key.

- - - - - 

Composite key = a key that needs 2 or more attributes. 

- one attribute cannot uniquely identify each row, 
- together they can uniquely identify each row, 
- that combination only shows up once. 

- - - - - 

Composite key = foreign key + (Surrogate/Natural) key.
or, 
Composite key = foreign key + foreign key. 
so, 
Composite key = combination of attributes that forms a primary key.
